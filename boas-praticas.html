Boas Práticas Profissionais

Dicas e padrões para escrever código limpo e acessível.

Semântica & Acessibilidade

  • Use landmarks (header, nav, main, footer) para orientar leitores de tela.
  • Forneça alt descritivo em imagens que transmitam informação.
  • Use labels associados a inputs e roles quando necessário.
  • Garanta foco visível e ordem de tabulação coerente.
<header>
  <nav><ul><li><a href="#">Início</a></li></ul></nav>
</header>
<main>
  <article><h1>Título</h1></article>
</main>

Responsividade

  • Adote mobile-first: escreva estilos base para mobile e aumente para telas maiores.
  • Use Flexbox e Grid para layouts adaptáveis.
  • Prefira unidades relativas (rem, %) e imagens responsivas (max-width:100%).
/* mobile-first */
.grid { display: grid; grid-template-columns: 1fr }
@media (min-width: 768px) { .grid { grid-template-columns: 1fr 1fr } }

Organização de CSS

  • Use convenção (ex: BEM) para previsibilidade.
  • Extraia variáveis CSS para cores, espaçamentos e tipografia.
  • Separe componentes em blocos e evite seletores muito específicos.
/* Exemplo BEM */
.card { }
.card__title { font-weight: 600 }
.card--highlight { border-color: var(--primary) }

Performance leve

  • Otimize imagens (WebP, compressão) e use width/height para evitar reflow.
  • Lazy-load de imagens abaixo da dobra.
  • Evite executar scripts pesados no carregamento inicial.
<img src="foto.webp" loading="lazy" alt="Descrição" width="800" height="600">

Versionamento (Git)

  • Commits pequenos e com mensagens descritivas (feat:, fix:, docs:).
  • Use branches para features e pull requests para revisão.
  • Inclua .gitignore e não comite credenciais.
git checkout -b feat/filtro-tecnologias
git add .
git commit -m "feat: filtro por categoria com botões"

Boas práticas de JavaScript

  • Prefira funções puras e evite efeitos colaterais quando possível.
  • Nomes claros para funções e variáveis; trate erros com try/catch.
  • Modularize código e documente responsabilidades dos módulos.
function soma(a,b) { return a + b } // função pura
